# 2026-05-07 Changelog

## 2026-05-07 - Settings Defaults, Language Packs, and Island Controls

The settings work was tightened so Settings owns app defaults while the translation island keeps temporary per-session state.

### Fixed

- Centered the Settings window on the active screen whenever it is opened from the translation island or menu bar.
- Clarified Default Translation Direction semantics: Settings saves the startup default, and every newly opened translation island initializes from that default.
- Kept language changes made inside the island local to the current island session. Swapping or selecting languages in the island no longer rewrites Settings defaults.
- Normalized legacy language identifiers such as `zh_CN`, `en_US`, and `en-Latn-US` into `zh-Hans` and `en-US`, keeping the default direction as Simplified Chinese to English.
- Removed the normal button chrome from translation action buttons while preserving press scale and short cyan click feedback.
- Replaced deprecated `NSSpeechSynthesizer` usage with `AVSpeechSynthesizer`.

### Language Management

- Reworked the Language settings pane as system Translation language-pack management rather than translation-direction management.
- Used Apple Translation supported languages and single-language availability status as the source of truth.
- Removed direction-specific labels such as current target and same-language translation warnings from the language-pack list.

### Known Follow-Up

- The pin button currently changes visual state, but the island can still collapse when the pointer leaves. Pin behavior needs a separate interaction pass.
