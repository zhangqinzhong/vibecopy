# 2026-05-11 Changelog

## 2026-05-11 - Clipboard Interaction Stability

This pass tightened clipboard-history interactions after the floating panel polish.

### Fixed

- Moved Escape handling to the clipboard window event path so the panel closes even when the list is empty or focus is inside the search field.
- Removed the visible per-row delete button; deletion is now keyboard-first with Delete or Forward Delete.
- Added selected-item replacement after delete: the next row is selected when available, otherwise the previous row is selected.
- Added a small delete transition so rows collapse into place instead of disappearing abruptly.
- Kept search text editing safe by only routing Delete to history deletion when the search field is not focused.
- Replaced the SwiftUI search field with an AppKit text field to avoid placeholder and cursor jitter when clicking history rows.
- Reworked row click handling so single-click selection is immediate and double-click paste remains available.
